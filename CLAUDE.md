# Websitenerstellung - SEO-optimiert mit KI-Unterstützung

## Projekt-Zweck
Dieses Projekt dient der Erstellung von SEO-optimierten Websites unter Nutzung modernster KI-Tools und Best Practices für "Generative Engine Optimization" (GEO) im Jahr 2026.

## Tech-Stack
- **Framework**: Next.js 15+ mit App Router
- **Styling**: Tailwind CSS
- **Sprache**: TypeScript
- **SEO-Analyse**: DataForSEO API
- **UI-Inspiration**: 21st.dev (nur Referenz, nicht Builder)
- **Dokumentation**: Context7 MCP

# SEO/GEO Best Practices 2026

## Die 4 Säulen der modernen SEO

### Säule 1: Intent-basierte Keyword-Strategie
Keywords werden nach Nutzerintention kategorisiert:

| Intent | Beschreibung | Beispiel |
|--------|--------------|----------|
| **Informational** | Nutzer sucht Wissen | "Was ist Next.js?" |
| **Commercial** | Nutzer vergleicht Optionen | "Next.js vs Nuxt Vergleich" |
| **Navigational** | Nutzer sucht spezifische Seite | "Next.js Dokumentation" |
| **Transactional** | Nutzer will kaufen/handeln | "Next.js Kurs kaufen" |

**Regel**: Jede Seite sollte auf EIN primäres Intent optimiert sein.

### Säule 2: Technische Exzellenz

#### Schema Markup (JSON-LD) - PFLICHT
Schema ist die "Muttersprache der KI". Jede Seite MUSS strukturierte Daten haben:

```typescript
// lib/schema.ts - Beispiel für Article Schema
export function generateArticleSchema(article: Article) {
  return {
    "@context": "https://schema.org",
    "@type": "Article",
    "headline": article.title,
    "description": article.description,
    "author": {
      "@type": "Person",
      "name": article.author
    },
    "datePublished": article.publishedAt,
    "dateModified": article.updatedAt
  }
}
```

**Wichtige Schema-Typen:**
- `Article` - für Blog-Posts und Artikel
- `FAQPage` - für FAQ-Seiten (höchster Wert für AI Overviews!)
- `Organization` - für Unternehmensseiten
- `Product` - für E-Commerce
- `LocalBusiness` - für lokale Unternehmen
- `BreadcrumbList` - für Navigation

#### Core Web Vitals Schwellenwerte
- **LCP (Largest Contentful Paint)**: < 2.5s (ideal < 1.5s)
- **INP (Interaction to Next Paint)**: < 200ms
- **CLS (Cumulative Layout Shift)**: < 0.1
- **Ladezeit für KI-Crawler**: < 1000ms

#### KI-Crawler Konfiguration (robots.txt)
```
# robots.txt - KI-Crawler NICHT blockieren!
User-agent: GPTBot
Allow: /

User-agent: Google-Extended
Allow: /

User-agent: anthropic-ai
Allow: /

User-agent: *
Allow: /
Disallow: /api/
Disallow: /admin/
```

### Säule 3: Content-Strategie für KI ("Fan-out" Optimierung)

LLMs nutzen "Fan-out" - eine Anfrage löst mehrere Unteranfragen aus. Optimiere dafür:

**Die 300-Zeichen-Regel:**
1. Jede H2-Überschrift als direkte Frage formulieren
2. Unmittelbar danach: Faktische Antwort in max. 300 Zeichen
3. Dann erst ausführliche Erklärung

**Beispiel:**
```markdown
## Was ist Next.js?

Next.js ist ein React-Framework für produktionsreife Webanwendungen mit
Server-Side Rendering, Static Site Generation und optimierter Performance.

[Hier folgt die ausführliche Erklärung...]
```

**Topical Authority aufbauen:**
- Alle verwandten Themen eines Clusters abdecken
- Interne Verlinkung zwischen verwandten Seiten
- "Pillar Page" + "Cluster Content" Struktur

### Säule 4: AI Overview & Knowledge Graph Optimierung

**Ziel**: In Google AI Overviews zitiert werden

**Strategie:**
1. Klare, faktische Aussagen machen
2. Quellen und Daten zitieren
3. FAQ-Schema für häufige Fragen
4. Expertise durch Author-Schema zeigen

# DataForSEO API Integration

## Authentifizierung
```typescript
// lib/dataforseo.ts
const DATAFORSEO_AUTH = Buffer.from(
  `${process.env.DATAFORSEO_LOGIN}:${process.env.DATAFORSEO_PASSWORD}`
).toString('base64');

async function callDataForSEO(endpoint: string, data: any) {
  const response = await fetch(`https://api.dataforseo.com${endpoint}`, {
    method: 'POST',
    headers: {
      'Authorization': `Basic ${DATAFORSEO_AUTH}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(data)
  });
  return response.json();
}
```

## API-Endpunkte Referenz

| Ziel | Endpunkt | Schlüssel-Parameter |
|------|----------|---------------------|
| KI-Sichtbarkeit prüfen | `POST /v3/serp/google/organic/live/advanced` | `"load_async_ai_overview": true` |
| Schema validieren | `POST /v3/on_page/task_post` | `"validate_micromarkup": true` |
| GPTBot simulieren | `POST /v3/on_page/task_post` | `"custom_user_agent": "Mozilla/5.0 (compatible; GPTBot/1.0)"` |
| Nutzerfragen finden | `POST /v3/dataforseo_labs/google/related_keywords/live` | `"keyword_intent": "informational"` |
| Marke in KI tracken | `POST /v3/ai_optimization/llm_mentions/search/live` | `"target": "[Markenname]"` |

## Fehlerbehandlung

| Code | Bedeutung | Aktion |
|------|-----------|--------|
| 40501 | Ungültiger Parameter | Payload gegen Docs prüfen |
| 50000 | Server-Fehler | Exponential Backoff (1s, 2s, 4s...) |
| Duplicate Crawl | Domain wird bereits gecrawlt | Auf vorherigen Task warten |

## Kosten-Awareness
- **LLM Mentions**: $0.10 pro Request + $0.001 pro Datenzeile
- **Rate Limit**: Max 2000 Requests/Minute (empfohlen: ~500/min)
- **Tipp**: Limit-Parameter nutzen, wenn nur Stichproben benötigt

## Neue Seite erstellen (SEO-konform)

1. **Keyword-Recherche** (DataForSEO Labs API)
   - Related Keywords abrufen
   - Intent analysieren
   - People Also Ask extrahieren

2. **Content-Struktur planen**
   - H1: Haupt-Keyword

## SEO-Audit durchführen

```typescript
// Vollständiges Audit einer bestehenden Seite
async function runSEOAudit(url: string) {
  // 1. OnPage Crawl
  const crawl = await callDataForSEO('/v3/on_page/task_post', [{
    target: url,
    validate_micromarkup: true,
    custom_user_agent: "Mozilla/5.0 (compatible; GPTBot/1.0)"
```
